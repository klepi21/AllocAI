One-line value proposition: "Pay once, get strategy + proof + actionable links."

Problem

What users struggle with today

Too many yield options

Users cannot easily choose which protocol/chain has the best stablecoin opportunity at a given time.

No trust in black-box advice

Recommendations without payment and proof tracking are hard to trust and hard to audit.

Execution friction

Users need direct links to execute strategy quickly, not generic descriptions.

No continuity

After refresh/device change, users lose context if run history is not stored on backend.

Solution

What AllocAI delivers now

  • Live data: Yield table from DeFiLlama with up to 2 best USDC opportunities per chain.
  • Paid run: Fixed `0.25 KITE` flow in "Run The Agent" modal.
  • Two payment modes: direct wallet KITE payment + AGENTIC x402 PAYMENTS via Passport.
  • Kite integrations: Lucid Bridge + Swap Aggregator + cross-chain USDC balance monitor.
  • Rich strategy output: recommendation, APR, execution steps, and 2Y/3Y/5Y compounding estimates.
  • Proof: on-chain proof tx hash shown to user with explorer link.
  • History: latest 5 runs per wallet, including status, tx links, and block metadata.

Function-by-Function Flow

How the product works under the hood

1) Yield discovery

`GET /api/yield` pulls DeFiLlama pools, filters USDC stable pools, ranks by TVL/APY, returns up to 2 per chain.

2) Paid execution

`POST /api/paid-data` supports direct wallet payment and AGENTIC x402 PAYMENTS with 402 challenge/settlement.

3) Lucid Bridge + swap aggregator

Integrated bridge and swap modules let users move and convert assets in-app across KITE-centered routes.

4) USDC balances + proof history

Cross-chain USDC balance endpoint and proof-aware run history show what the wallet has and what the agent executed.

5) Autonomous scheduler

VPS-triggered generic autonomous profile runs on cadence with secure tick secret and independent proof trail.

6) Guardrails + KPIs

Policy constraints (slippage/allocation/TVL/confidence/cooldown) plus KPI API for success-rate and latency visibility.
